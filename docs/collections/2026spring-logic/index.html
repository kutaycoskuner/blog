Introduction

Logic is a branch of mathematics and philosophy that studies the principles of valid reasoning and inference. It provides a formal framework for analyzing statements, arguments, and their relationships in terms of truth and structure. The fundamental objects in logic include propositions, logical connectives, and quantifiers, which are used to build precise statements about objects and their properties.

One of the central ideas of logic is that complex reasoning can be reduced to simple, well-defined rules of manipulation of symbols. By representing statements in a formal language, one can determine whether arguments are valid, inconsistent, or satisfiable using systematic procedures. Concepts such as truth tables, logical equivalence, proof systems, and predicate logic form the core tools of this field.

Logic plays a crucial role in many areas of science, mathematics, and computer science. In mathematics, it is used to construct rigorous proofs and define the foundations of mathematical structures. In computer science, logic is essential for designing algorithms, programming languages, and verification systems. It underlies areas such as artificial intelligence, databases, circuit design, and formal verification of software and hardware systems. Philosophy uses logic to analyze arguments and clarify reasoning, while linguistics applies it to study the structure of language.

Because of its fundamental role in reasoning and computation, logic is considered one of the key foundations of both theoretical and applied disciplines in modern science and technology. **** 1. Notation

In logic and mathematics, precise notation is essential to describe objects and their relationships clearly and unambiguously. In this chapter, we introduce the basic notation for sets and fundamental set operations.

Definition: Set

die Menge

A set is a well-defined collection of distinct objects, called elements.

There are two common ways to describe a set:

Extensional definition (Aufzählende Darstellung)
A set is described by explicitly listing its elements.

A={1,2,3}A = \{1, 2, 3\}A={1,2,3}

Intensional definition (beschreibende Darstellung)
A set is described by a property that its elements satisfy.

A={x∣x>0,x∈N}A = \{x \mid x > 0, x \in \mathbb{N}\}A={x∣x>0,x∈N}

Equality of Sets

Two sets are equal if they contain exactly the same elements.

A=B  ⟺  (∀x:x∈A⇔x∈B)A = B \iff (\forall x: x \in A \Leftrightarrow x \in B)A=B⟺(∀x:x∈A⇔x∈B)

Order and repetition do not matter in sets:

{1,2,3}={3,2,1}={1,1,2,3}\{1,2,3\} = \{3,2,1\} = \{1,1,2,3\}{1,2,3}={3,2,1}={1,1,2,3}

Logical operators

Logical Notation

logic operators / logische Operatoren

Logic uses symbols called logical operators (logische Operatoren) to combine statements and build more complex expressions from simpler ones.

A basic indivisible statement is called an atomic proposition (Atom).
An atom is a statement that is either true or false.

Examples of atoms:

  • “It rains”
  • “2 is even”
  • "x>0x > 0x>0"

Atoms can be combined using logical operators.

Negation (Negation)
Reverses the truth value of a statement.

¬A\neg A¬A

Meaning: “not AAA”

Example:

If A=A =A= “It rains”, then:

¬A="It does not rain"\neg A = \text{"It does not rain"}¬A="It does not rain"

Conjunction (Konjunktion)
Logical AND.

A∧BA \land BA∧B

True only if both AAA and BBB are true.

Example:

"It rains"∧"It is cold"\text{"It rains"} \land \text{"It is cold"}"It rains"∧"It is cold"

Disjunction (Disjunktion)
Logical OR.

A∨BA \lor BA∨B

True if at least one of AAA or BBB is true.

Implication (Implikation)
“If … then …”

A⇒BA \Rightarrow BA⇒B

If AAA is true, then BBB must also be true.

Example:

x>2⇒x>1x > 2 \Rightarrow x > 1x>2⇒x>1

Equivalence (Äquivalenz)
“If and only if”

A⇔BA \Leftrightarrow BA⇔B

True exactly when both statements have the same truth value.

Example:

x is even⇔x mod 2=0x \text{ is even} \Leftrightarrow x \bmod 2 = 0x is even⇔xmod2=0

Relations

Relation (Relation)

eine Relation

A relation is a set of ordered pairs describing a connection between elements.

For sets SSS and TTT, a relation RRR from SSS to TTT is defined as:

R⊆S×TR \subseteq S \times TR⊆S×T

This means every element of RRR is a pair (s,t)(s,t)(s,t) where:

s∈S,t∈Ts \in S,\quad t \in Ts∈S,t∈T

and:

(s,t)∈R(s,t) \in R(s,t)∈R

Often, instead of writing:

(s,t)∈R(s,t) \in R(s,t)∈R

we write:

sRtsRtsRt

or equivalently:

R(s,t)R(s,t)R(s,t)

These notations mean exactly the same thing:
”sss is related to ttt under relation RRR.”

Example:

Let:

S={1,2,3},T={2,4,6}S = \{1,2,3\},\quad T = \{2,4,6\}S={1,2,3},T={2,4,6}

Define:

R={(1,2),(2,4),(3,6)}R = \{(1,2),(2,4),(3,6)\}R={(1,2),(2,4),(3,6)}

Then:

(2,4)∈R(2,4) \in R(2,4)∈R

can also be written as:

2R42R42R4

or:

R(2,4)R(2,4)R(2,4)

Subset Relation (Teilmengenrelation)

Subset is itself a relation between sets.

A⊆BA \subseteq BA⊆B

means:

∀x:  x∈A⇒x∈B\forall x:\; x \in A \Rightarrow x \in B∀x:x∈A⇒x∈B

Every element of AAA is also an element of BBB.

Properties of Relations

Let

R⊆S×TR \subseteq S \times TR⊆S×T

be a relation from set SSS to set TTT.

We write:

sRtsRtsRt

or equivalently:

R(s,t)R(s,t)R(s,t)

to mean:

(s,t)∈R(s,t) \in R(s,t)∈R

The following properties describe how elements of SSS and TTT are connected.

Left-total (linkstotal)

A relation is left-total if every element in SSS is related to at least one element in TTT.

∀s∈S,  ∃t∈T:  sRt\forall s \in S,\; \exists t \in T:\; sRt∀s∈S,∃t∈T:sRt

Every element on the left side has a connection.

Example:

S={1,2,3},T={a,b}S=\{1,2,3\}, \quad T=\{a,b\}S={1,2,3},T={a,b} R={(1,a),(2,b),(3,a)}R=\{(1,a),(2,b),(3,a)\}R={(1,a),(2,b),(3,a)}

This relation is left-total because every element in SSS appears at least once.

Not left-total:

R={(1,a),(3,b)}R=\{(1,a),(3,b)\}R={(1,a),(3,b)}

Here element 222 has no relation.

Right-total (rechtstotal)

A relation is right-total if every element in TTT is related to by at least one element in SSS.

∀t∈T,  ∃s∈S:  sRt\forall t \in T,\; \exists s \in S:\; sRt∀t∈T,∃s∈S:sRt

Every element on the right side is reached.

Example:

S={1,2,3},T={a,b}S=\{1,2,3\}, \quad T=\{a,b\}S={1,2,3},T={a,b} R={(1,a),(2,a),(3,b)}R=\{(1,a),(2,a),(3,b)\}R={(1,a),(2,a),(3,b)}

Both aaa and bbb are related to, so the relation is right-total.

Not right-total:

R={(1,a),(2,a)}R=\{(1,a),(2,a)\}R={(1,a),(2,a)}

Element bbb is never reached.

Left-unique (linkseindeutig) / Injective (injektiv)

A relation is left-unique if one element in TTT cannot be related to two different elements in SSS.

∀s1,s2∈S,  ∀t∈T:\forall s_1,s_2 \in S,\; \forall t \in T:∀s1​,s2​∈S,∀t∈T: (s1Rt∧s2Rt)⇒s1=s2(s_1Rt \land s_2Rt) \Rightarrow s_1=s_2(s1​Rt∧s2​Rt)⇒s1​=s2​

Each right element has at most one left predecessor.

Example:

R={(1,a),(2,b),(3,c)}R=\{(1,a),(2,b),(3,c)\}R={(1,a),(2,b),(3,c)}

Left-unique.

Not left-unique:

R={(1,a),(2,a)}R=\{(1,a),(2,a)\}R={(1,a),(2,a)}

Element aaa has two predecessors.

This violates left-uniqueness.

Right-unique (rechtseindeutig)

A relation is right-unique if one element in SSS cannot be related to two different elements in TTT.

∀s∈S,  ∀t1,t2∈T:\forall s \in S,\; \forall t_1,t_2 \in T:∀s∈S,∀t1​,t2​∈T: (sRt1∧sRt2)⇒t1=t2(sRt_1 \land sRt_2) \Rightarrow t_1=t_2(sRt1​∧sRt2​)⇒t1​=t2​

Each left element has at most one image.

Example:

R={(1,a),(2,b),(3,a)}R=\{(1,a),(2,b),(3,a)\}R={(1,a),(2,b),(3,a)}

Right-unique.

Not right-unique:

R={(1,a),(1,b)}R=\{(1,a),(1,b)\}R={(1,a),(1,b)}

Element 111 points to two different elements.

This violates right-uniqueness.

Functional (funktional)

A relation is called functional if it is:

  • left-total (linkstotal)
  • right-unique (rechtseindeutig)

So:

∀s∈S,  ∃! t∈T:  sRt\forall s \in S,\; \exists !\, t \in T:\; sRt∀s∈S,∃!t∈T:sRt

Every element in SSS has exactly one image in TTT.

This is exactly a function (Funktion).

Example:

R={(1,a),(2,b),(3,a)}R=\{(1,a),(2,b),(3,a)\}R={(1,a),(2,b),(3,a)}

Every left element has exactly one output.

So RRR is functional.

Surjective (surjektiv)

A function is surjective if every element in TTT is reached.

Equivalent to right-total:

∀t∈T,  ∃s∈S:  sRt\forall t \in T,\; \exists s \in S:\; sRt∀t∈T,∃s∈S:sRt

Example:

S={1,2,3},T={a,b}S=\{1,2,3\}, \quad T=\{a,b\}S={1,2,3},T={a,b} R={(1,a),(2,b),(3,a)}R=\{(1,a),(2,b),(3,a)\}R={(1,a),(2,b),(3,a)}

Both aaa and bbb are reached.

Therefore surjective.

Not surjective:

R={(1,a),(2,a),(3,a)}R=\{(1,a),(2,a),(3,a)\}R={(1,a),(2,a),(3,a)}

bbb is never reached.

Bijective (bijektiv)

A function is bijective if it is:

  • injective (injektiv) = left-unique
  • surjective (surjektiv) = right-total
  • functional

Equivalently:

  • every element in SSS maps to exactly one element in TTT
  • every element in TTT is reached exactly once

Example:

S={1,2,3},T={a,b,c}S=\{1,2,3\}, \quad T=\{a,b,c\}S={1,2,3},T={a,b,c} R={(1,a),(2,b),(3,c)}R=\{(1,a),(2,b),(3,c)\}R={(1,a),(2,b),(3,c)}

Every left element maps exactly once.
Every right element is reached exactly once.

Therefore the relation is bijective.

A bijection creates a perfect one-to-one correspondence (eine eindeutige Zuordnung).

Cardinality of Sets (Mächtigkeit der Menge)

The cardinality (Mächtigkeit) of a set describes the size of a set, meaning the number of elements it contains.

For a set AAA, cardinality is written as:

∣A∣|A|∣A∣

Example:

If

A={1,2,3,4}A=\{1,2,3,4\}A={1,2,3,4}

then:

∣A∣=4|A|=4∣A∣=4

If

B={a,b,c}B=\{a,b,c\}B={a,b,c}

then:

∣B∣=3|B|=3∣B∣=3

For the empty set:

∣∅∣=0|\emptyset|=0∣∅∣=0

Two sets have the same cardinality if they contain the same number of elements.

Example:

A={1,2,3},B={a,b,c}A=\{1,2,3\},\quad B=\{a,b,c\}A={1,2,3},B={a,b,c}

Then:

∣A∣=∣B∣=3|A|=|B|=3∣A∣=∣B∣=3

Cardinality can be finite or infinite.

Finite example:

∣{x,y,z}∣=3|\{x,y,z\}|=3∣{x,y,z}∣=3

Infinite example:

∣N∣=∞|\mathbb{N}|=\infty∣N∣=∞

In set theory, infinite sets can also have different cardinalities (unterschiedliche Mächtigkeiten).

Remark on Infinite Cardinalities

The comparison of cardinalities of infinite sets (for example N\mathbb{N}N, Q\mathbb{Q}Q, R\mathbb{R}R) is based on formal set-theoretic results.

At this point, I do not include a detailed justification of these results, since the constructions and proofs (e.g. diagonal arguments, explicit enumerations) are not yet fully understood.

Only the final classifications are noted:

  • N\mathbb{N}N: infinite set
  • Q\mathbb{Q}Q: infinite set
  • R\mathbb{R}R: infinite set with a different structure of infinity

A deeper treatment requires additional set theory tools.
