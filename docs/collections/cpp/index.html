Inheritance

Published: Thu, 30 Jan 2025 Revision: Thu, 30 Jan 2025, v 0.1 Language: This content is in English.

Inheritance is a fundamental concept in object-oriented programming (OOP) that allows one class (the derived class) to inherit the properties and behaviors (methods) of another class (the base class). This concept promotes code reuse, modularity, and the creation of class hierarchies.

Key Concepts of Inheritance

Base Class (Parent Class)

The class whose properties and methods are inherited by another class.
class Animal { ... };

Derived Class (Child Class)

The class that inherits from the base class.
class Dog : public Animal { ... };

Access Specifiers

Inheritance can be controlled using access specifiers

  • public : Public and protected members of the base class remain public and protected in the derived class.
  • protected : Public and protected members of the base class become protected in the derived class.
  • private : Public and protected members of the base class become private in the derived class.

Types of Inheritance

// # Single Inheritance: A derived class inherits from only one base class.
class Animal { ... };
class Dog : public Animal { ... };
// # Multiple Inheritance: A derived class inherits from more than one base class.
class Animal { ... };
class Mammal { ... };
class Dog : public Animal, public Mammal { ... };
// # Multilevel Inheritance: A derived class inherits from another derived class.
class Animal { ... };
class Mammal : public Animal { ... };
class Dog : public Mammal { ... };
// # Hybrid Inheritance: A combination of multiple and multilevel inheritance.
class Animal { ... };
class Mammal : public Animal { ... };
class Bird : public Animal { ... };
class Bat : public Mammal, public Bird { ... };

Constructor and Destructor in Inheritance

  • The base class constructor is called before the derived class constructor.
  • The derived class destructor is called before the base class destructor.

Function overloading

The derived class can override methods of the base class to provide specific implementations.

class Animal {
public:
    void speak() { cout << "Animal sound" << endl; }
};
class Dog : public Animal {
public:
    void speak() { cout << "Woof!" << endl; } // Overriding
};

Practical Use of Inheritance

Code Reusability
Inheritance allows you to reuse code from the base class, reducing redundancy and improving maintainability. Example: If multiple classes share common attributes (e.g., Animal class with name and age), you can define them once in the base class.
Polymorphism
Inheritance enables polymorphism, where a derived class object can be treated as a base class object. Example: A Dog object can be treated as an Animal object, allowing for flexible and extensible code.
Extensibility
You can extend the functionality of a base class without modifying it. Example: Add a new method fetch() to the Dog class without changing the Animal class.
Real-World Modeling
Inheritance helps model real-world relationships (e.g., a Dog is an Animal).

Cons

Tight Coupling
Changes in the base class can affect all derived classes, leading to fragile code.
Complexity
Deep inheritance hierarchies can make the code harder to understand and maintain.
Misuse
Overusing inheritance can lead to overly complex designs (e.g., "god objects").
Multiple Inheritance Issues
Multiple inheritance can lead to ambiguity (e.g., the "diamond problem").

Meta Programming

Published: Thu, 30 Jan 2025 Revision: Thu, 30 Jan 2025, v 0.1 Language: This content is in English.

Template meta-programming (TMP) is a technique where templates are used to perform computations, generate code, or make decisions at compile time. It leverages the C++ compiler to evaluate expressions and generate code before the program runs.

Idea

  • Templates are not just for generic programming (writing code that works with any type). They can also be used to perform compile-time computations and code generation.
  • The compiler evaluates templates and generates code based on the rules of template instantiation.

- Templates

  • Templates are the foundation of meta-programming in C++. They allow you to write generic code that works with any data type.
  • There are two types of templates:
    • Function Templates  : Define generic functions.
    • Class Templates     : Define generic classes.
// # Function template example
#include <iostream>
#include <string>

// Renamed to avoid conflict with std::max
template <typename T>
T my_max(const T& a, const T& b) {
    return (a > b) ? a : b;
}

int main() {
    // Example with integers
    int intMax = my_max(10, 20);
    std::cout << "Max of 10 and 20: " << intMax << std::endl;

    // Example with doubles
    double doubleMax = my_max(3.14, 2.71);
    std::cout << "Max of 3.14 and 2.71: " << doubleMax << std::endl;

    // Example with strings
    std::string strMax = my_max(std::string("apple"), std::string("banana"));
    std::cout << "Max of 'apple' and 'banana': " << strMax << std::endl;

    return 0;
}

- Template specialization

  • Specialization allows you to define specific implementations of a template for particular types.
  • This is useful when the generic implementation doesn’t work for certain types.
// # Template specialization
#include <iostream>

// Primary template (generic case)
template <typename T>
void print(T value) {
    std::cout << "General template: " << value << std::endl;
}

// Specialization for `int`
template <>
void print<int>(int value) {
    std::cout << "Specialized template for int: " << value << std::endl;
}

int main() {
    print(3.14);  // Uses the general template
    print(42);    // Uses the specialized template for int

    return 0;
}

- Type Traits

Type traits are a way to introspect and manipulate types at compile-time.

They are often used in template meta-programming to make decisions based on the properties of types.

// # Type traits example
#include <iostream>
#include <type_traits>

// Function template using type traits
template <typename T>
void printType() {
    if (std::is_integral<T>::value) {
        std::cout << "Integral type\n";
    } else {
        std::cout << "Non-integral type\n";
    }
}

int main() {
    printType<int>();      // Integral type
    printType<double>();   // Non-integral type
    printType<char>();     // Integral type
    printType<float>();    // Non-integral type
    printType<long>();     // Integral type
    printType<std::string>(); // Non-integral type

    return 0;
}

- constexpr

constexpr is a keyword in C++ that indicates that the value of a variable or function can be evaluated at compile time. This allows for optimizations and can improve performance by moving computations from runtime to compile time. It was introduced in C++11 and has been expanded in subsequent standards (C++14, C++17, and C++20).

Key Concepts

constexpr Variables
Variables whose values are known at compile time.
constexpr Functions
Functions that can be evaluated at compile time if their arguments are constant expressions.
constexpr Constructors
Allow the creation of constexpr objects.
// # Example constexpr function
#include <iostream>

constexpr int factorial(int n) {
return (n <= 1) ? 1 : n \* factorial(n - 1);
}

int main() {
constexpr int value = factorial(5); // Computed at compile time
std::cout << "Factorial of 5: " << value << std::endl;
return 0;
}

Explanation

factorial(5) is computed at compile time because it is marked as constexpr. The result is stored in the constexpr variable value.

When to Use constexpr

Compile-Time Calculations
Use constexpr for calculations that can be done at compile time to improve runtime performance.
Constant Values
Use constexpr for values that are known at compile time and will not change.
Template Metaprogramming
constexpr is often used in template metaprogramming to perform computations at compile time.
Optimization
Use constexpr to enable compiler optimizations and reduce runtime overhead.

Pros

Performance
Moves computations to compile time, reducing runtime overhead.
Safety
Ensures that values are constant and cannot be modified.
Readability
Makes it clear that a value or function is intended to be evaluated at compile time.
Metaprogramming
Enables powerful compile-time programming techniques.

Cons

Restrictions
Functions and constructors must adhere to strict rules (e.g., no I/O, no dynamic memory allocation).
Complexity
Can make code harder to understand for beginners.
Compiler Support
Older compilers may not fully support constexpr features.

- Variadic Templates

  • Variadic templates allow you to define templates that accept a variable number of arguments.
  • This is useful for creating generic functions or classes that work with any number of arguments.
// # Variadic template example
#include <iostream>

// Base case: Stop recursion when no arguments remain
void print() {
    std::cout << std::endl;
}

// Variadic template function
template <typename First, typename... Rest>
void print(First first, Rest... rest) {
    std::cout << first << " ";
    print(rest...); // Recursive call with remaining arguments
}

int main() {
    print(1, 2.5, "Hello", 'A'); // Works with different types
    return 0;
}

Practical Use of Template Meta-Programming

Generic Libraries
Template meta-programming is widely used in libraries like the Standard Template Library (STL) to create generic containers (e.g., std::vector, std::map) and algorithms (e.g., std::sort, std::find).
Compile-Time Computations
Template meta-programming allows you to perform computations at compile-time, which can lead to more efficient runtime performance. Example: Calculating factorials, Fibonacci numbers, or other mathematical operations at compile-time.
Type Safety and Code Reusability
Templates enable you to write type-safe code that can be reused across different data types without duplicating code.
Domain-Specific Languages (DSLs)
Template meta-programming can be used to create embedded DSLs within C++, allowing for more expressive and concise code in specific domains.

Why It Could Not Be Solved Other Way

Runtime Overhead
Without templates, you would need to write separate functions or classes for each data type, leading to code duplication and increased runtime overhead.
Lack of Flexibility
Traditional approaches (e.g., function overloading) are limited in their ability to handle arbitrary types and compile-time logic.
Performance
Template meta-programming allows for compile-time optimizations that are not possible with runtime techniques.

Pros

Code Reusability
Write once, use with any type.
Performance
Compile-time computations reduce runtime overhead.
Type Safety
Templates ensure type correctness at compile-time.
Flexibility
Supports complex compile-time logic and type manipulation.

Cons

Complexity
Template meta-programming can be difficult to understand and debug.
Compile-Time Errors
Errors in templates can lead to cryptic and hard-to-debug messages.
Code Bloat
Excessive use of templates can lead to increased binary size due to code generation for each type.

Functor and Lambda

Published: Thu, 30 Jan 2025 Revision: Thu, 30 Jan 2025, v 0.2 Language: This content is in English.

Functors

A functor (or function object) in C++ is an object that can be used as if it were a function. This is achieved by overloading the operator() for a class. Functors are more powerful than regular functions because they can maintain state between calls and can be customized through their constructors. They could be passed as parameters to other functions.

Key Concepts:

Overloading operator()
A class becomes a functor by defining the operator() method.
Stateful Behavior
Functors can store data as member variables, allowing them to maintain state across multiple calls.
Customization
Functors can be initialized with data, making them highly flexible.

Example

#include <iostream>
#include <vector>
#include <algorithm>
#include <cmath>

struct CompareByProximity {
    int threshold;

    CompareByProximity(int t) : threshold(t) {}  // Constructor to store extra parameter

    bool operator()(int a, int b) const {
        return std::abs(a - threshold) < std::abs(b - threshold);
    }
};

int main() {
    std::vector<int> nums = {10, 3, 7, 15, 2};
    int threshold = 8;

    std::sort(nums.begin(), nums.end(), CompareByProximity(threshold));  // Works

    for (int n : nums) std::cout << n << " ";
}

With Normal Function

bool compare(int a, int b, int threshold) {  // Error: std::sort expects a function with two parameters only
    return std::abs(a - threshold) < std::abs(b - threshold);
}

std::sort(nums.begin(), nums.end(), compare);  // ❌ Won't work: expects a function with (int, int)

Explanation

A normal function cannot be used because std::sort requires a callable with exactly two parameters, while a functor (or lambda) can encapsulate additional state.

When to use functors

Custom Behavior
When you need custom logic that depends on state or parameters.
STL Algorithms
When working with STL algorithms like std::sort, std::count_if, etc.
Stateful Functions
When you need a function-like object that maintains state between calls.
Performance
Functors can be inlined by the compiler, potentially offering better performance than function pointers.

Pros

Stateful
Can maintain state between calls.
Customizable
Can be initialized with parameters for flexible behavior.
Performance
Often faster than function pointers due to inlining.
Compatibility
Works seamlessly with STL algorithms.

Cons

Verbosity
Requires defining a class, which can be overkill for simple tasks.
Complexity
Can make code harder to read for beginners.
Object Overhead
Requires creating an object, which may introduce slight overhead.

Additions

In modern C++ (C++11 and later), lambdas are often preferred over functors for simple tasks because they are more concise. However, functors are still useful for complex logic or when reusability is important.

Lambda functions

Lambdas, introduced in C++11, are a concise way to define anonymous functions (i.e., functions without a name) directly in your code. They are particularly useful for short, one-off functions that are passed as arguments to higher-order functions like STL algorithms. Lambdas can capture variables from their surrounding scope, making them highly flexible and expressive.

Key Concepts

  1. Syntax: A lambda has the following general syntax:
[capture](parameters) -> return_type { body }
Capture
Specifies which variables from the surrounding scope are accessible inside the lambda.
Parameters
The input arguments for the lambda (similar to function parameters).
Return Type
Optional; the compiler can often deduce it automatically.
Body
The code that executes when the lambda is called.

  1. Capture Clauses:

    • []: Capture nothing.
    • [&]: Capture all variables by reference.
    • [=]: Capture all variables by value.
    • [var]: Capture var by value.
    • [&var]: Capture var by reference.
    • [this]: Capture the this pointer (for member functions).

  2. Mutable Lambdas: By default, variables captured by value are const inside the lambda. Use the mutable keyword to modify them.

  3. Return Type: If the body contains multiple return statements or complex logic, you may need to explicitly specify the return type using -> return_type.

Example

#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    std::vector<int> nums = {1, 2, 3, 4, 5};

    std::for_each(nums.begin(), nums.end(), [](int n) {
        std::cout << n << " ";
    });

    return 0;
}

Explanation

This lambda simply takes an integer n and prints it, making it a clean and modern alternative to loops.

Why Use a Lambda Here?

Concise
No need to define a separate function.
Inline Readability
Keeps logic close to where it's used.
No Boilerplate
Avoids defining a named function or functor.

Type Deduction

Published: Thu, 30 Jan 2025 Revision: Thu, 30 Jan 2025, v 0.1 Language: This content is in English.

Type deduction in C++ refers to the compiler’s ability to automatically infer the type of a variable or expression based on the context, without the programmer having to explicitly specify it. This feature simplifies code, enhances flexibility, and reduces redundancy.

- auto keyword

The auto keyword allows the compiler to automatically deduce the type of a variable from its initializer.

auto x = 5;   // x is deduced to be of type int
auto y = 3.14; // y is deduced to be of type double

- decltype Keyword

decltype is used to explicitly deduce the type of an expression without evaluating it.

int a = 10;
decltype(a) b = a;  // b is deduced to be of type int

- Function Return Type Deduction

C++14 introduced the ability to deduce the return type of a function using auto.

auto add(int a, int b) {
    return a + b; // deduces return type to be int
}

- Template Type Deduction

Templates can use type deduction for function or class templates.

template <typename T>
void print(T value) {
    std::cout << value << std::endl;
}

int main() {
    print(42); // T is deduced to be int
}

Practical Use

  1. Why Type Deduction is Useful

Code Simplification: Without type deduction, you’d have to explicitly declare types, which can be repetitive and error-prone, especially for complex types like iterators, lambdas, or template classes.

Generic Code: Type deduction is crucial when writing generic code (e.g., template functions or classes). It allows the compiler to infer the correct type for any provided argument without needing to write specific type overloads.

Without type deduction, you’d need to explicitly write multiple overloads for different types, leading to code duplication and clutter.

  1. Why It Could Not Be Solved Another Way

Without type deduction, the programmer would have to manually specify types in many places, which could lead to cumbersome, error-prone, and less flexible code.

For example, without auto, you would have to explicitly write complex iterator types for container elements:

std::vector<int>::iterator it = vec.begin();  // long and error-prone
auto it = vec.begin();  // much more concise and clear

Pros

Code Conciseness: It eliminates the need to explicitly declare types, leading to more readable and concise code.

Maintainability: Reduces code duplication, making it easier to modify, refactor, or extend code (e.g., changing a container type doesn’t require changing variable types everywhere).

Improved Generic Programming: Facilitates generic code where the types are automatically inferred, enabling better code reuse.

Cons

Loss of Explicitness: The type may be unclear to someone reading the code, especially for more complex expressions, which can reduce code clarity.

Compiler Errors: The compiler’s type deduction might not always work as expected, leading to confusing or complex error messages, particularly in complex template-based code.

Overuse of auto: If overused, it can hide the underlying type, leading to ambiguity, especially for function return types.

Virtual Destructors

Published: Fri, 31 Jan 2025 Revision: Fri, 31 Jan 2025, v 0.1 Language: This content is in English.

A virtual destructor ensures that the destructor of a derived class is called when an object is deleted through a base class pointer. It prevents resource leaks and undefined behavior.

Polymorphism & Base-Class Pointer Behavior

  • If a base class has a non-virtual destructor, deleting a derived class object via a base class pointer will cause only the base class destructor to execute, leading to resource leaks.
  • A virtual destructor ensures that the derived class destructor runs first, followed by the base class destructor.

Virtual Table (vtable) Mechanism

  • When a class has a virtual function (including a virtual destructor), the compiler generates a vtable for dynamic dispatch.
  • The destructor call goes through this vtable to ensure the correct order of destruction.
// # example without virtual destructor
#include <iostream>

class Base {
public:
    ~Base() { std::cout << "Base Destructor\n"; }
};

class Derived : public Base {
public:
    ~Derived() { std::cout << "Derived Destructor\n"; }
};

int main() {
    Base* obj = new Derived();
    delete obj; // Undefined behavior! Only Base's destructor runs.
    return 0;
}
// # output incorrect leading to leaks
// Here, the Derived class destructor never executes, potentially causing memory/resource leaks.
Base Destructor
// # Corrected Version Using a Virtual Destructor
#include <iostream>

class Base {
public:
    virtual ~Base() { std::cout << "Base Destructor\n"; }
};

class Derived : public Base {
public:
    ~Derived() { std::cout << "Derived Destructor\n"; }
};

int main() {
    Base* obj = new Derived();
    delete obj; // Proper cleanup!
    return 0;
}

// # Correct Output
Derived Destructor
Base Destructor

Virtual Table

Published: Fri, 31 Jan 2025 Revision: Fri, 31 Jan 2025, v 0.1 Language: This content is in English.

The virtual table (vtable) is a mechanism used in C++ to implement dynamic (runtime) polymorphism. It ensures that the correct function is called when a virtual function is invoked through a base class pointer.

What is a vtable?

  • A vtable is a table of function pointers maintained per class that contains entries for all virtual functions in that class.
  • Each class with virtual functions has a single vtable.

vptr (Virtual Table Pointer)

  • Each object of a class that has at least one virtual function contains a hidden vptr (virtual table pointer).
  • This vptr points to the class’s vtable.
  • The compiler automatically manages vptr and updates it when needed.

How it Works

  • When a virtual function is called through a base class pointer or reference, the function call is resolved at runtime using the vtable.
  • The program looks up the appropriate function address in the vtable and calls it.
// # Without Virtual Functions (Compile-Time Binding)
#include <iostream>

class Base {
public:
    void show() { std::cout << "Base class\n"; }
};

class Derived : public Base {
public:
    void show() { std::cout << "Derived class\n"; }
};

int main() {
    Base* obj = new Derived();
    obj->show(); // Calls Base::show() due to early binding
    delete obj;
}
// # output
// Even though obj is pointing to a Derived object, the base class function is called because compile-time binding is used.
Base class
// # With Virtual Functions (Runtime Binding Using vtable)
#include <iostream>

class Base {
public:
    virtual void show() { std::cout << "Base class\n"; } // Virtual function
    virtual ~Base() {} // Virtual destructor (important for cleanup)
};

class Derived : public Base {
public:
    void show() override { std::cout << "Derived class\n"; }
};

int main() {
    Base* obj = new Derived();
    obj->show(); // Calls Derived::show() due to vtable lookup
    delete obj;
}
// # output
Derived class

Here’s what happens behind the scenes:

  • The Derived class overrides show(), so its vtable entry replaces Base::show().
  • obj->show() triggers a vtable lookup, calling Derived::show().

Practical Uses of vtable

Runtime Polymorphism
Enables dynamic dispatch, allowing function calls to be determined at runtime. Example: GUI frameworks, where different widgets (buttons, labels) override a common draw() function.

Abstract Base Classes (Interfaces)
Pure virtual functions (= 0) ensure derived classes provide implementations. Example: std::ostream in C++ uses virtual functions for polymorphic output handling.

Plugin Architectures
Applications use base class interfaces and dynamically load different implementations at runtime.

Multithreading

Published: Fri, 31 Jan 2025 Revision: Fri, 31 Jan 2025, v 0.1 Language: This content is in English.

Multithreading in C++ allows a program to execute multiple tasks concurrently by running them in separate threads within the same process. It helps in utilizing multi-core processors efficiently.

  • Thread: A lightweight unit of execution that runs independently.
  • std::thread: A C++ standard library class that provides support for creating and managing threads.
  • Synchronization
    • Mutex (std::mutex): Used to prevent multiple threads from accessing shared resources simultaneously. (Mutual exclusivity)
    • Lock (std::unique_lock, std::lock_guard): Ensures proper locking and unlocking of mutexes.
    • Condition Variables (std::condition_variable): Helps in synchronization by allowing threads to wait for some condition to be met.
  • Thread Safety Ensuring that shared data is accessed correctly in a multithreaded environment.
  • Atomic Operations (std::atomic): Provides operations that are inherently thread-safe without requiring locks.
  • Thread Communication: Threads can communicate via shared variables, message passing, or synchronization primitives.
  • Thread Lifecycle
    • Creation: Using std::thread to start a new thread.
    • Joining (join()): Waiting for a thread to complete execution.
    • Detaching (detach()): Running a thread independently without waiting for completion.

Use Cases

Parallel Computing
Running independent computations simultaneously (e.g., matrix multiplication, image processing).
Asynchronous Tasks
Using std::async for background computations.
GUI Applications
Keeping UI responsive while performing background operations.
I/O Operations
Handling multiple network requests, file operations, etc.
Gaming and Simulations
Running physics calculations, AI, and rendering separately.

Cons

Complexity
Harder to debug and maintain.
Race Conditions
If multiple threads access shared data incorrectly.
Deadlocks
Occurs when two or more threads wait indefinitely for each other.
Increased Memory Usage
Each thread has its own stack, leading to more memory consumption.

Named member initialization

Published: Fri, 31 Jan 2025 Revision: Fri, 31 Jan 2025, v 0.1 Language: This content is in English.

Named member initialization is a feature in C++ that allows you to initialize specific data members of a class or struct by name during object construction. This is particularly useful for clarity and flexibility, especially when dealing with classes or structs that have many members or default values.

Syntax

Named member initialization uses the . or -> operator to specify which member to initialize:

struct Point {
    int x;
    int y;
};

Point p = { .x = 10, .y = 20 };

Order of Initialization:

Members must be initialized in the order they are declared in the class or struct. Skipping members or changing the order will result in a compilation error.

Default Values

Members not explicitly initialized will retain their default values (if provided) or be default-initialized.

Summary
Named member initialization allows you to initialize specific members of a class or struct by name, improving code clarity and flexibility.

Introduced in C++20, it is particularly useful for structs with many members or when partial initialization is desired.

While it offers many advantages, it is limited to aggregates and requires C++20 or later.
